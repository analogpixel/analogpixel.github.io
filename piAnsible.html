Configure the PI for Ansible

Before Ansible can connect, you need to create a user, setup sudo and some ssh keys for the authentication; that's all you really need for ansible to connect and start causing trouble.

Configure a simple Ansible Server

Ansible doesn't really need a server, it can run on anything that has python, but I like to keep everything reproduceable so I created a docker container to hold all the ansible information.

# build with docker build --rm -t ansible .
FROM ubuntu:latest

RUN apt-get update
run apt-get install -y python python-pip libssl-dev iputils-ping openssh-client
run pip install ansible

run mkdir /ansible
run mkdir -p /etc/ansible

run echo "192.168.0.132" > /etc/ansible/hosts
env ANSIBLE_HOST_KEY_CHECKING=False

WORKDIR /ansible

---
- hosts: all
  remote_user: ansible
  become: true

  tasks:
  - name: install udhcpd
    apt: name=dhcp3-server state=latest

  - name: copy udhcpd config
    copy:
      src: dhcpd.conf
      dest: /etc/dhcp/dhcpd.conf
      owner: root
      group: root
      mode: 0644

  - name: copy over webapp
    copy:
      src: webapp/
      dest: /webapp/
      owner: root
      group: root

  - name: copy over interfaces adhoc
    copy:
      src: interfaces_adhoc
      dest: /etc/network/interfaces_adhoc
      owner: root
      group: root
      mode: 0644

  - name: copy over interfaces infra
    copy:
      src: interfaces_infra
      dest: /etc/network/interfaces_adhoc
      owner: root
      group: root
      mode: 0644
